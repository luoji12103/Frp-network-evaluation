## 2. 职责划分

### 根目录
放项目说明和启动入口。`main.py` 只负责加载配置、调用 orchestrator、导出结果。

### config/
放所有可配置内容。不能把 host、port、阈值、ssh 信息写死在代码里。

### bin/
放一键脚本。脚本职责是安装依赖、激活环境、启动程序，不负责业务逻辑。

### probes/
所有探测逻辑都放这里，包括系统命令执行、输出解析、统计聚合。任何对 ping、iperf3、tcp connect 的细节处理都应在这里完成。

### agents/
作为执行端包装层，负责“接收任务并调用 probes”。不应该持有复杂业务逻辑。

### controller/
作为总控层，负责编排执行顺序、远程调用、结果收集和场景切换。

### exporters/
负责最终产出。不要在 exporter 中再执行探针。

### results/
每次 run 的输出目录。

### logs/
用于放运行日志和调试信息。

## 3. 为什么要这样分

这样拆分的价值在于：

- probe 可单测
- orchestrator 可替换
- agent 可本地执行也可远程执行
- exporter 不影响探测逻辑
- Codex 更容易按模块生成，而不是把所有逻辑塞进一个大脚本
