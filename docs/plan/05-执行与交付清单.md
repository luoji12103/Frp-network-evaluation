# 执行与交付清单

## 1. 推荐实施顺序

### 阶段 1：初始化仓库
- 建目录
- 写 `requirements.txt`
- 写配置样例
- 写 README 和 bootstrap 脚本

### 阶段 2：实现基础 probe
- `ProbeResult`
- `run_cmd`
- `ping.py`
- `tcp_handshake.py`
- `system_probe.py`

### 阶段 3：实现吞吐 probe
- `throughput.py`
- iperf3 JSON 解析
- forward/reverse 支持

### 阶段 4：实现 agent
- `agent_client.py`
- `agent_relay.py`
- `agent_server.py`

### 阶段 5：实现 SSH 与 orchestrator
- `ssh_exec.py`
- `orchestrator.py`
- 场景执行顺序

### 阶段 6：实现 exporter
- JSON
- CSV
- HTML

### 阶段 7：实现带载退化场景
- 压测并发 TCP 采样
- 生成膨胀指标

## 2. 交付物清单

项目第一版至少应交付：

1. 源码目录
2. 示例配置
3. README
4. bootstrap 脚本
5. 一次完整测试结果样例
6. 文档说明

---

## 3. 最低验收命令

建议至少支持：

```bash
bash bin/bootstrap_mac.sh
bash bin/bootstrap_linux.sh
python main.py --topology config/topology.example.yaml --thresholds config/thresholds.example.yaml
```

---

## 4. 报告中必须出现的字段

- 路径名
- probe 名
- success
- RTT 均值/P95
- jitter
- 丢包
- TCP connect 均值/P95
- 吞吐上下行
- 带载膨胀
- 系统 CPU/内存
- 异常项列表
- 最终瓶颈结论
